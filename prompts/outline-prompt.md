# 骨子生成プロンプト

以下の条件で記事の骨子（アウトライン）を生成してください。

## ステップ0: プロンプトマッピング（骨子生成の前に必ず実施）

骨子を作成する前に、以下の手順で読者の検索意図を深掘りする。

1. 対象キーワード `{{keyword}}` について、読者がAI（ChatGPTなど）に質問する場合の言い回しを **3〜5パターン** 想定して列挙する
   - 例: 「doda ワークポート どっちがいい」→「dodaとワークポートって結局どっちに登録すべき？」「第二新卒で転職エージェント使うならdodaかワークポートどっちがおすすめ？」など
2. 列挙したAI質問パターンを確認し、骨子内の **FAQセクション** がそれらの言い回しに直接答えられる構成になっているか検証する
3. カバーできていない言い回しがあれば、FAQ候補を追加・修正して対応する

> **このステップの目的:** AI検索時代において、読者がLLMに投げる自然な質問文にFAQが直接ヒットするようにし、AI引用（AIO）されやすい骨子を設計する。

## 入力情報
- キーワード: {{keyword}}
- カテゴリ: {{category}}
- ターゲット: 第二新卒（24歳前後、転職経験0〜1回）でIT/Web業界へのキャリアチェンジを検討している人

```markdown

### [h3見出し1-1]
### [h3見出し1-2]

## [h2見出し2]（PREP: Point）

### [h3見出し2-1]
### [h3見出し2-2]

...

## よくある質問

- Q: [質問文1]
- Q: [質問文2]
- Q: [質問文3]
```

## 比較記事の構造ルール（cta_agentsが2つ以上の場合）

比較記事では、各比較対象をh3見出しで区切り、以下の形式を **必ず** 含めること。
既存の比較表やタイプ別レコメンドの構成は維持したうえで、各サービス紹介に追加する。

```markdown
### [サービス名]

[100〜200字の概要文]

**こんな人におすすめ**: [1行でターゲット像をまとめる]

**メリット**
- [メリット1]
- [メリット2]
- [メリット3]
（3〜4個）

**デメリット**
- [デメリット1]
- [デメリット2]
（2〜3個）
```

※ `####` 見出しは使わず、`**メリット**` / `**デメリット**` の太字ラベルで統一すること。

## ルール
1. h2は3〜5個。各h2の冒頭にPREPのPoint（結論文）を1行で記載すること
2. h3は各h2の下に1〜3個
3. 「よくある質問」セクションにFAQ候補を3つ以上含める（質問文のみでOK）
4. 検索意図を満たす構成にする（情報収集→比較検討→行動喚起の流れ）
5. CTA挿入ポイントを `[CTA挿入ポイント]` で明示する
6. ステップ0のプロンプトマッピングを実施してからFAQを確定すること
